El login permite identificar a un usuario antes de darle acceso a las funciones de una aplicación. Gracias a la autenticación se puede controlar quién puede consultar, crear o modificar información. Algunas rutas deben estar protegidas porque contienen datos privados o funciones administrativas. De esta manera se evita el acceso no autorizado. No deben exponerse públicamente contraseñas, documentos de identidad, teléfonos, direcciones, tokens, claves de API ni información sensible de la comunidad.
